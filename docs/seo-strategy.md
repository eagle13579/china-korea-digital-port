### 3.4 全站通用 — BreadcrumbList + LocalBusiness

在每个页面添加面包屑导航结构化数据：

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {"@type": "ListItem", "position": 1, "name": "首页", "item": "https://go-aiport.com"},
    {"@type": "ListItem", "position": 2, "name": "合规检查", "item": "https://go-aiport.com/compliance-check.html"}
  ]
}
```

### 3.5 多语言 hreflang 建议

在 JSON-LD 中通过 `@language` 和多语言链接提供语言信息，但更推荐在 HTML `<head>` 中使用带 `hreflang` 的 `<link>` 标签（已在 sitemap.xml 中实现）。

## 5. 外链和内容营销策略

### 5.1 外链建设策略

#### 中文站点（百度优化）
1. **行业门户合作**：在中国跨境电商、国际贸易类网站（如雨果网、亿邦动力网、亿欧网）发布行业分析文章

3. **跨境电商论坛**：在福步外贸论坛、阿里巴巴外贸圈等板块发布专业帖
4. **行业媒体**：与36氪、动点科技等科技媒体合作，报道AI+出海创新项目
5. **百科引用**：在百度百科"韩国KC认证""中韩FTA"等词条中引用（需权威性）

#### 韩国站点（Naver 优化）
1. **Naver Blog/Post**：创建 Naver 博客，发布韩语原创内容（韩国 SEO 最重要的非官方渠道）
2. **Naver Knowledge iN**：在 지식iN 回答韩国企业进入中国市场相关问题
3. **韩国电商协会**：与 한국전자상거래협회 等组织合作获取 backlink
4. **韩国创业媒体**：在 Platum、beSuccess 等创业媒体发布报道
5. **카카오톡 채널**：建立 KakaoTalk 频道传播内容

#### 英文站点（Google 优化）
1. **LinkedIn 专业内容**：CEO/团队发布行业洞见文章
2. **跨境贸易博客**：在 TradeReady、GlobalTrade 等行业博客投稿
3. **Medium**：发布技术博客和行业分析
4. **ProductHunt**：上线产品获取初始曝光和 backlink

## 6. 百度 vs Naver 搜索差异及策略

### 6.1 核心差异对比

| 维度 | 百度 (中国) | Naver (韩国) | Google (全球) |
|------|------------|--------------|---------------|
| 市场地位 | 70%+ 市场份额 | 60%+ 市场份额 | 韩国 ~30%，中国受限 |
| 爬虫友好度 | 较慢收录，需百度站长验证 | 需 Naver 注册，支持 RSS | 最快收录 |
| 关键词匹配 | 精确匹配权重较高 | 语义理解较强 | 语义理解最强 |
| 外链权重 | 高权重，但质量要求严 | 中权重，Naver Blog 影响大 | 高权重 |
| 图片搜索 | 重要流量来源 | Naver 图片搜索也很重要 | 中重要 |
| 移动端优化 | 重要（MIP/AMP支持） | 重要（Naver App 生态） | 重要（Core Web Vitals） |
| 本地化要求 | ICP备案、服务器位置影响 | 韩语内容需自然地道 | 语言地道性 |
| 社交媒体关联 | 百度知道、百家号 | Naver Blog、카페 | SNS 信号 |

### 6.2 百度搜索优化策略

1. **ICP 备案**：确保服务器在中国有备案，否则 Baidu 可能不收录
2. **百度站长平台**：提交 sitemap，使用百度自动推送 JS
3. **移动端优化**：百度优先收录移动端页面，确保响应式设计
4. **标题规范**：标题控制在 28 个中文字符以内，核心关键词前置
5. **关键词密度**：控制在 2%-8%，自然融入
6. **百家号联动**：在百度百家号发布文章，建立百度生态内的内容据点
7. **页面加载速度**：百度对加载速度敏感，确保 ≤ 3 秒

### 6.3 Naver 搜索优化策略

1. **Naver 注册**：在 Naver Search Advisor 注册网站并提交 sitemap
2. **韩语内容质量**：避免机器翻译韩语，需由母语者撰写或严格审校
3. **Naver Blog 发布**：定期在 Naver Blog 发布韩语文章（Naver 搜索结果中博客权重很高）
4. **Naver Knowledge iN**：积极参与问答，建立权威性
5. **页面结构**：使用清晰的标题层级（H1 → H2 → H3）
6. **Naver Webtoon 等 Naver 生态**：利用 Naver 旗下产品获取额外曝光
7. **카카오톡 分享**：优化社交分享标签（Open Graph），方便在 KakaoTalk 分享
8. **Naver 专有结构化数据**：支持 Naver 特有的搜索特性（如知识面板）

### 6.4 技术 SEO 通用建议

#### 页面加载性能
- 目标：Lighthouse 评分 ≥ 90
- Core Web Vitals：LCP < 2.5s，FID < 100ms，CLS < 0.1
- 使用 WebP 格式图片，启用浏览器缓存
- 关键 CSS 内联，非关键 CSS 延迟加载

#### 多语言 SEO
```html
<!-- 每个页面 head 中需包含 -->
<link rel="alternate" hreflang="zh-CN" href="https://go-aiport.com/page.html" />
<link rel="alternate" hreflang="ko-KR" href="https://go-aiport.com/page.html?lang=ko-KR" />
<link rel="alternate" hreflang="en" href="https://go-aiport.com/page.html?lang=en" />
<link rel="alternate" hreflang="x-default" href="https://go-aiport.com/page.html" />
```

#### Open Graph / Twitter Card
```html
<meta property="og:title" content="..." />
<meta property="og:description" content="..." />
<meta property="og:image" content="https://go-aiport.com/og-image.png" />
<meta property="og:url" content="https://go-aiport.com/page.html" />
<meta name="twitter:card" content="summary_large_image" />
```

#### Canonical URL
确保所有多语言页面使用正确的 `canonical` 标签指向本语言版本。
