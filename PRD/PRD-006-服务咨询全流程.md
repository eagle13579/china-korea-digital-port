## 1. 产品概述

### 1.1 一句话定位
打通从用户浏览网站 → 提交咨询 → 销售跟进 → 报价 → 成交的完整转化漏斗，消除每个环节的断裂点。

### 1.2 目标用户
| 用户类型 | 描述 |
|---------|------|
| 韩企CEO/代表理事 | 决策者，浏览网站后希望快速了解方案和报价 |
| 韩企法务/合规负责人 | 执行者，需要咨询具体合规问题后再决策 |
| 中国合作伙伴 / BD渠道 | 中间人，需要推荐客户后跟踪进展 |

### 1.3 痛点分析
当前已实现的咨询触点（来自代码审计）：
- **联系表单**：POST `/api/v1/contact` → 写入 contacts 表 ✅
- **定价咨询**：POST `/api/v1/pricing/inquiry` → 写入 pricing_inquiries 表 ✅
- **演示预约**：POST `/api/v1/demo` → 写入 demo_requests 表 ✅
- **服务邀请**：POST `/api/v1/service-inquiry` → 写入 service_inquiries 表 ✅
- **管理后台**：`/api/v1/admin/leads` 支持线索查看和状态管理 ✅

核心痛点：
1. **漏斗断裂**：用户提交咨询后，销售团队没有系统化的「跟进看板」——全靠手动查数据库
2. **状态流转不完整**：leads 表虽有 status 字段，但admin后台缺少「自动通知销售」和「超时提醒」
3. **没有报价生成能力**：用户在咨询后，销售需要手动计算报价、手动回复邮件
4. **多渠道咨询无统一视图**：四个咨询表相互独立，admin后台虽有聚合但缺少按「企业维度」的合并视图
5. **没有成交闭环**：从「咨询→报价→成交」没有数字化追踪，北极星指标「成交数」不可衡量

### 1.4 北极星关联
| 北极星维度 | 预期健康度提升 | 说明 |
|:----------|:--------------:|:-----|
| C商业化 | +8% | 直接提升咨询到成交的转化率 |
| B数字员工 | +3% | 通过服务邀请流程反向验证员工匹配度 |

## 2. 功能规格

### 2.1 用户故事

#### US-001：管理员查看并筛选所有线索
**作为** 销售团队（英招/鹿蜀）
**我想要** 在一个页面看到所有渠道的咨询线索（联系、定价、演示、服务邀请），按状态/时间/企业名筛选
**以便于** 知道哪些客户需要优先跟进

**验收条件：**
- admin后台默认展示所有线索的聚合列表（4张表 union）
- 支持按表类型筛选（contacts/demo_requests/pricing_inquiries/service_inquiries）
- 支持按状态筛选（new/contacted/qualified/negotiating/lost 等）
- 支持按时间范围筛选和排序
- 每条线索显示：企业名、联系人、邮箱、电话、咨询类型、来源渠道、提交时间、当前状态
- 支持模糊搜索（企业名/联系人/邮箱）

#### US-002：线索状态流转与备注
**作为** 销售团队
**我想要** 跟进线索时能更新其状态并添加备注
**以便于** 团队协作和时间线追溯

**验收条件：**
- 点击线索进入详情页
- 状态可变更（流转路径见下方流程图）
- 每次状态变更记录时间戳和操作人
- 支持添加跟进备注（文本，最长2000字），每条备注有时间戳
- 备注不可删除（可编辑最近一条），确保审计日志完整性

**状态流转图：**
```
new → contacted → qualified → negotiating → closed_won
  ↘               ↘                         → closed_lost
      → lost        → closed_lost
```

#### US-003：自动销售通知
**作为** 销售团队成员
**我想要** 新线索提交后能收到即时通知（站内/邮件/飞书）
**以便于** 0延迟响应客户

**验收条件：**
- admin后台顶部显示「新线索」红色徽章（未读数）
- 可选发送邮件通知到销售团队邮箱
- 可选集成飞书/钉钉Webhook通知（配置化，不在MVP强制）
- 通知内容包含：客户企业名、联系人、咨询类型、提交时间

#### US-004：报价单生成
**作为** 销售人员（英招）
**我想要** 在跟进线索时能快速生成报价单（包含选中的套餐和定制项）
**以便于** 缩短从咨询到报价的周期

**验收条件：**
- 在线索详情页支持「生成报价」按钮
- 报价模板包含：企业名称、日期、选中的方案（免费初评/深度方案¥9,800/年订阅¥58,000）、数量、有效期（默认7天）
- 支持添加自定义报价项（如额外服务、折扣）
- 报价单可以预览和下载为PDF
- 报价单状态追踪：已发送 / 已查看 / 已接受 / 已拒绝

#### US-005：报价→成交闭环
**作为** 销售团队和管理层
**我想要** 追踪报价单从发送到成交的完整链路
**以便于** 计算转化率并优化销售流程

**验收条件：**
- 报价单「已接受」后自动关联线索状态变更为「closed_won」
- 报价单「已拒绝」后自动关联线索状态变更为「closed_lost」
- 后台统计：线索总数 / 跟进中 / 成交数 / 成交率 / 平均成交周期
- 数据可用于北极星指标「C商业化」的量化评估

#### US-006：定价页→咨询→报价一体化
**作为** 韩企潜在客户
**我想要** 在定价页选择方案后直接发起咨询，并获得针对性报价
**以便于** 快速决策

**验收条件：**
- pricing.html 已经实现 `openPricingInquiry(planType)` 函数 ✅（代码已存在第237行）
- 表单已包含 plan_type 隐藏字段 ✅
- 提交到 `/api/v1/pricing/inquiry` ✅
- 新增：提交成功后，系统自动发送报价单到客户邮箱（邮件模板需开发）
- 新增：销售后台自动创建一条「待跟进报价」记录

## 3. 验收标准

### 3.1 产品层验收

#### P-AC-001：线索聚合展示
- **Given** 管理员登录admin后台
- **When** 访问线索管理页面
- **Then** 看到所有4张表的线索统一展示
- **And** 可按表类型、状态、时间筛选
- **And** 支持模糊搜索企业名/联系人

#### P-AC-002：状态流转
- **Given** 管理员在某条线索详情页
- **When** 变更状态（如 new → contacted）
- **Then** 状态变更成功，页面刷新显示新状态
- **And** 系统记录变更时间和操作人

#### P-AC-003：跟进备注
- **Given** 管理员在线索详情页
- **When** 添加备注内容并提交
- **Then** 备注显示在备注列表中，包含时间和操作人
- **And** 备注内容不包含XSS风险（textContent处理）

#### P-AC-004：报价生成
- **Given** 管理员在线索详情页点击「生成报价」
- **When** 填写报价项并提交
- **Then** 生成报价单并显示预览
- **And** 支持导出为文本/PDF格式

### 3.2 技术层验收

#### T-AC-001：性能
- 线索聚合查询响应 ≤ 500ms（4表 UNION ALL）
- 报价单生成 ≤ 2s（含PDF生成）
- API 分页支持（默认20条/页）

#### T-AC-002：安全
- 所有admin接口需要token认证（现有 `_require_auth` 机制 ✅）
- 备注内容做XSS过滤
- 报价单不暴露敏感的内部成本信息

#### T-AC-003：数据一致性
- 报价单状态变更同步更新线索状态（报价 accepted → lead closed_won）
- 线索删除时触发的关联数据清理规则需明确定义

### 6.2 审查意见

3. 报价单PDF建议增加水印「机密文件，仅限收件人查阅」
