# PRD-003 SEO基础

## 1. 产品概述

### 问题
首页缺少SEO优化，搜索引擎无法有效索引网站内容。Google、Bing、Naver的中韩关键词搜索排名为零，潜在客户无法通过搜索引擎发现平台。

### 目标用户
- **韩企合规负责人**：在Naver/Google搜索"中国合规"、"韩企中国法人设立"等关键词时找到本平台
- **中国合作伙伴**：在百度/必应搜索"韩企合规"、"中韩跨境法务"时发现本平台

### KPI
- 3个月内自然搜索带来100+ UV/月
- sitemap提交后，Google/Bing索引覆盖度 > 80%
- robots.txt和sitemap.xml正常响应（200 OK）

## 2. 功能规格

### US-001: 页面SEO元数据
所有页面 `<head>` 添加以下标签：
- `meta name="description"` — 中韩双语描述（通过data-lang切换）
- `meta property="og:title"` 和 `og:description`、`og:image`、`og:url`
- `meta name="twitter:card"`、`twitter:title`、`twitter:description`

### US-002: 中韩双语SEO Tags
每个页面通过 `data-lang-key` 属性实现中韩双语SEO标签切换：
- 中文页面：`lang="zh-CN"`
- 韩文页面：`lang="ko-KR"`
- 双语交替标签 `<link rel="alternate" hreflang="zh-CN" />` 和 `hreflang="ko-KR"`

### US-003: /robots.txt
允许爬取所有公开页面，拒绝爬取以下敏感路径：
```
Disallow: /admin/
Disallow: /api/
Disallow: /private/
```

### US-004: /sitemap.xml
列出所有公开页面URL，包含：
- 首页
- 服务介绍页
- 合规指南页
- 关于我们页
- 联系方式页
- 每个URL的中韩双语alternate标记

### US-005: JSON-LD结构化数据
首页添加 JSON-LD：
- `@type: Organization` — 公司名称、Logo、描述（中韩双语）
- `@type: WebSite` — 网站名称、URL、alternate名称
- 中韩双语alternate链接

## 3. 验收标准

### 产品层
- [ ] 每个页面 `<head>` 有完整的SEO标签（description、OG、Twitter Card）
- [ ] `/robots.txt` 返回200，内容正确
- [ ] `/sitemap.xml` 返回200，所有链接有效
- [ ] JSON-LD结构化数据通过 Google Rich Results 测试
- [ ] 中韩双语交替标签完整

### 技术层
- [ ] SEO标签不影响页面加载速度（无额外阻塞请求）
- [ ] 中韩语言切换后，`lang` 属性正确变更
- [ ] SEO标签不暴露敏感信息（管理路径、API端点等）

## 4. 技术方案

### 前端改造（仅需修改静态文件）

1. **修改 `index.html` 的 `<head>`**
   - 添加 `meta description`
   - 添加 OG tags（og:title、og:description、og:image、og:url）
   - 添加 Twitter Card tags
   - 添加 JSON-LD 结构化数据脚本块
   - 添加 `link rel="alternate"` hreflang 标签

2. **添加 `/robots.txt`**
   - 创建项目根目录静态文件
   - 内容：Allow公开路径 / Disallow敏感路径

3. **添加 `/sitemap.xml`**
   - 创建静态文件方式（如项目为纯静态站点）
   - 或通过后端路由动态生成（如项目有后端）

4. **修改 `js/main.js`（如需中韩SEO tags切换逻辑）**
   - 语言切换时同步更新 `meta description` 和 OG tags
   - 切换 `lang` 属性

### 不需修改
- 后端代码（本项目为纯前端改造）
- 数据库结构
- API接口

## 5. 技术风险

| 风险等级 | 风险描述 | 缓解措施 |
|---------|---------|---------|
| 低 | 纯前端改造，不影响后端逻辑 | 所有改动限定在HTML/JS层面 |
| 中 | 机器人爬取时触发API调用导致性能问题 | 已确认：本站为全静态页面，无API依赖 |
| 低 | SEO tags中包含的URL写死导致部署后失效 | 使用相对路径或配置化URL注入 |
| 低 | 双语切换时SEO标签不同步 | JS中增加lang-change事件监听，同步更新SEO meta |

## 8. 不做清单

本PRD明确界定以下内容不在本次实施范围内：

- ❌ **不做百度推广投放** — 不涉及百度SEM/竞价排名
- ❌ **不做付费SEO** — 不购买付费外链或SEO服务
- ❌ **不做内容营销策略** — 不涉及博客/白皮书/行业报告等内容创作
- ❌ **不做外链建设** — 不申请外部网站链接交换
- ❌ **不做社交媒体SEO** — 不涉及微博/小红书/抖音等平台内容
- ❌ **不做Naver付费广告** — 不涉及Naver搜索广告投放
- ❌ **不做关键词广告** — 不涉及Google Ads/Naver Ads等PPC投放
