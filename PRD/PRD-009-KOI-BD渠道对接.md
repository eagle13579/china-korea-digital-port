## 1. 产品概述

### 1.1 一句话定位
为KOI（韩国Opinion Leader）/BD渠道合作伙伴提供专属推广链接、线索追踪和分佣结算功能，通过渠道网络规模化获客。

### 1.2 目标用户
| 用户类型 | 描述 |
|---------|------|
| 浩然（合作伙伴） | 已确认的BD渠道，推广中韩出海服务 |
| 儒商联盟 | 中介/顾问网络，推荐韩企客户 |
| 其他KOI/BD伙伴 | 在韩国的行业KOL、咨询公司、律所推荐人 |

### 1.3 痛点分析
- **当前无渠道识别**：所有咨询线索（联系/演示/定价/服务邀请）均没有「来源渠道」字段（虽有 `source` 字段但默认为 'website'，未用于渠道追踪）
- **无法归因**：无法知道某个客户是哪个渠道合作伙伴推荐的
- **无分佣机制**：合作伙伴推荐了客户后，没有系统化的佣金结算流程
- **无推广素材**：合作伙伴缺少标准化的推广物料（链接、海报、文案）

### 1.4 北极星关联
| 北极星维度 | 预期健康度提升 | 说明 |
|:----------|:--------------:|:-----|
| C商业化 | +6% | 渠道获客补充自然流量和付费流量 |
| S战略 | +4% | 建立BD渠道网络，提升市场覆盖 |

## 2. 功能规格

### 2.1 用户故事

#### US-001：渠道合作伙伴专属推广链接
**作为** 渠道合作伙伴（浩然/儒商联盟）
**我想要** 获得一个带参数的专属推广链接（如 `?ref=haoran`）
**以便于** 我推荐的客户访问网站时能被识别为我带来的

**验收条件：**
- 每个渠道合作伙伴有唯一的 `ref_code`（如 `haoran`, `rushang`）
- 推广链接格式：`https://go-aiport.com?ref={ref_code}`
- 用户通过推广链接访问后，`ref_code` 通过 localStorage 持久化（有效期30天）
- 用户通过推广链接提交任何咨询表单（contact/demo/pricing/service-inquiry）时，自动携带 `ref_code`
- 所有咨询表的数据表中 `source` 字段记录渠道来源（如 `ref:haoran`）

#### US-002：渠道线索看板
**作为** 运营团队（鹿蜀）
**我想要** 在admin后台按渠道合作伙伴查看所有线索及状态
**以便于** 评估每个渠道的贡献和转化效果

**验收条件：**
- admin后台新增「渠道分析」tab
- 展示每个渠道合作伙伴的：线索数、咨询数、成交数、成交金额、佣金估算
- 支持按时间范围筛选
- 支持导出渠道数据报表

#### US-003：佣金结算
**作为** 财务人员（蠪侄）
**我想要** 根据成交订单自动计算渠道佣金
**以便于** 定期向合作伙伴结算

**验收条件：**
- 佣金比例可配置（按合作伙伴类型或成交套餐）
- 建议佣金比例（需财务确认）：
  - 免费初评（¥0）：无佣金
  - 深度方案（¥9,800）：10%~15%
  - 年订阅（¥58,000）：15%~20%
- 成交确认后自动生成佣金记录
- 支持标记「已结算」状态
- 支持导出佣金报表给合作伙伴

#### US-004：渠道推广素材库
**作为** 渠道合作伙伴
**我想要** 获得标准化的推广素材（文案模板、海报、链接）
**以便于** 我在社交媒体和线下一对一推广时使用

**验收条件：**
- 每个渠道合作伙伴登录后可见专属素材包
- 素材包含：标准推广文案（中韩双语）、推荐话术、简单海报、专属二维码
- 素材可下载（PDF/图片格式）
- （v2）素材支持在线编辑（替换企业名）

### 2.3 API 设计

#### 2.3.2 数据模型

class ChannelPartner(BaseModel):

#### 2.3.3 数据库新增表

```sql
CREATE TABLE IF NOT EXISTS channel_partners (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    ref_code TEXT UNIQUE NOT NULL,
    contact_person TEXT,
    contact_email TEXT,
    contact_phone TEXT,
    commission_rate REAL DEFAULT 0.1,
    status TEXT DEFAULT 'active',
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE IF NOT EXISTS commissions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    partner_id INTEGER NOT NULL,
    lead_table TEXT,
    lead_id INTEGER,
    deal_amount REAL,
    commission_rate REAL,
    commission_amount REAL,
    status TEXT DEFAULT 'pending',  -- pending | settled | cancelled
    settled_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (partner_id) REFERENCES channel_partners(id)
);
```

## 3. 验收标准

### 3.1 产品层验收

#### P-AC-001：渠道参数识别
- **Given** 用户访问 `https://go-aiport.com?ref=haoran`
- **When** 页面加载完成
- **Then** `ref_code` 写入 localStorage
- **And** 用户提交任何表单时，`source` 字段包含该 `ref_code`

#### P-AC-002：渠道线索归因
- **Given** admin查看渠道分析
- **When** 查看「浩然」渠道
- **Then** 展示该渠道带来的所有线索及状态

#### P-AC-003：佣金计算
- **Given** 某渠道带来的线索成交
- **When** 手动触发佣金计算
- **Then** 佣金记录生成（成交金额 × 佣金比例）

### 3.2 技术层验收

#### T-AC-001：参数持久化
- `ref_code` 在 localStorage 中保留30天
- 超过30天后无新访问则清除
- ref_code 不随语言切换或页面跳转而丢失

#### T-AC-002：安全
- ref_code 只允许字母、数字、下划线、短横线（正则过滤）
- 不允许 XSS 注入
- admin接口需token认证

#### T-AC-003：性能
- 渠道数据统计响应 ≤ 1s
- 不影响现有表单提交性能

## 4. 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 渠道参数被恶意篡改（伪造ref_code） | 低 | 低 | 佣金结算前需人工审核成交记录 |
| localStorage被用户清除导致归因丢失 | 低 | 中 | 30天有效期内用户再次访问可重新关联 |
| 渠道佣金计算争议 | 中 | 低 | 佣金规则在合作协议中明确，系统保留完整审计日志 |

## 7. 法律审查

### 7.1 审查结论
**法律审查通过。签字：獬豸（法律合规官），2026-05-05**

### 7.2 审查意见
| 审查维度 | 状态 | 说明 |
|---------|------|------|
| 渠道合作协议 | ❌ 缺失 | 建议在功能上线前准备标准《渠道合作伙伴协议》模板 |
| 佣金结算法律基础 | ✅ 合规 | 佣金需在合作协议中明确约定比例和结算方式 |
| 数据共享合规 | ✅ 合规 | 渠道合作伙伴可查看其推荐线索的摘要数据，不可查看其他渠道数据 |

### 7.3 法律建议
1. 上线前需准备《渠道合作伙伴协议》模板，包含：合作范围、佣金比例、结算周期、争议解决、保密条款、终止条件
2. 渠道合作伙伴查看的数据应限制为「自己推荐的线索」，不可跨渠道查看
3. 佣金结算建议使用对公账户转账，保留完整的税务凭证

## 8. 不做清单

| 事项 | 原因 | 后续可能 |
|------|------|---------|
| ❌ 渠道合作伙伴自助注册 | 当前数量少，手动创建即可 | v3可考虑 |
| ❌ 实时佣金计算（成交即结算） | MVP阶段先手动触发 | v2可自动化 |
| ❌ 渠道分级（不同等级不同佣金比例） | 当前合作伙伴数量少 | v2可考虑 |
| ❌ 自动生成推广海报 | 设计成本高，当前手动提供 | v2 |
| ❌ 合作伙伴培训资料/考试系统 | 非MVP范围 | 不考虑 |
| ❌ 与第三方联盟平台集成（如ShareASale） | 超出规模 | v3可考虑 |
| ❌ 渠道合作伙伴之间的竞争性报价 | 当前无此需求 | 不考虑 |
