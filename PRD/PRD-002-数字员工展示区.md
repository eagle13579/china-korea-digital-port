## 1. 产品概述

### 1.1 问题陈述

中韩出海数智港的核心价值是「6位AI数字合规官」组成的合规产品线。但当前网站完全没有展示这6位数字员工的信息，用户无法感知到「谁在为我服务」，也无法形成信任锚点。

**核心问题**：客户在面对AI产品时，最大的疑虑是「这东西靠谱吗？」。如果我们把数字员工展示得像「招聘市场上的候选人」——有名字、有履历、有擅长领域、有经验数据、有成本对比——客户的信任感会大幅提升。

**竞争格局对比：**
- Deel 在首页和 /team 页面展示各国专家团队，让客户感知「有人服务」
- 猎聘/Deel的候选人卡片模式：姓名+职位+技能标签+经验年限+好评率
- 我们：数字合规官作为核心资产，却「隐身」了——这是产品层面的致命缺陷

**关键洞察**：韩企客户（尤其是CEO/法务负责人）的决策逻辑是「我需要一个懂xx的人」而不是「我需要看一个品牌故事」。招聘风格的展示格式能直击其决策心智。

### 1.2 产品愿景

让数字员工展示区成为中韩出海数智港的「人才市场」。每位合规官不是冷冰冰的算法，而是一个有履历、有专长、有「性价比」的数字员工。用户浏览 /team 页面后，能够：

1. 像浏览猎聘候选人一样快速找到自己需要的合规方向
2. 产生「这个员工正好能帮我解决问题」的匹配感
3. 看到价格对比后产生「太划算了」的冲动
4. 明确下一步行动（面试/邀请服务）

### 1.3 目标用户

| 用户类型 | 描述 |
|---------|------|
| 韩企CEO/代表理事 | 决策者，关注「找谁做」「多少钱」「靠不靠谱」 |
| 韩企法务/合规负责人 | 执行者，关注「他懂不懂我的具体业务场景」 |
| 中国合作伙伴 | 中间人，需要向韩企展示「我们的团队比真人便宜10倍」 |

### 1.4 成功指标（KPI）

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| /team 页面访问量占比 | 占全站PV ≥ 15% | GA4 |
| 页面平均停留时长 | ≥ 60秒（招聘浏览式行为） | GA4 |
| 从 /team 到「邀请服务」的转化率 | ≥ 10% | 漏斗分析 |
| 6位员工中至少4位被点击查看详情 | 点击率 ≥ 65% | 事件埋点 |
| 薪资对比标签的点击/悬停互动率 | ≥ 30% | 事件埋点 |

## 2. 功能规格

### 2.1 用户故事

#### US-001：浏览数字员工团队（招聘集市）

**作为** 韩企合规负责人
**我想要** 像在猎聘网上浏览候选人一样，看到每位数字员工的履历和专长
**以便于** 快速判断「哪个人能解决我当前的问题」

**验收条件：**
- 每位员工展示：头像、中韩双语姓名、职位头衔、级别、能力清单（3-5个带勾选标记）、经验数据
- 每位员工卡片上显示「月薪 ¥xxx」及「对标真人成本」对比标签
- 页面顶部为招聘市场风格的Hero区文案
- 页面支持中韩双语切换
- 页面在桌面端/平板/手机端均正常渲染

#### US-002：查看数字员工详情（候选人简历）

**作为** 韩企CEO
**我想要** 点击某位员工查看他的完整履历和能力详情
**以便于** 判断他是否适合我的企业需求，并产生「请Ta为我服务」的冲动

**验收条件：**
- 点击卡片弹出详情模态框（或跳转至 /team/{slug}）
- 详情页展示：完整能力列表、擅长场景示例、经验数据、薪资对比
- 详情页底部有「邀请Ta为我服务」按钮
- ❌ 不展示神兽名称、灵魂来源、大师对标

#### US-003：触发服务邀请

**作为** 潜在客户
**我想要** 在了解某位数字员工后快速发起服务邀请/咨询
**以便于** 开始使用合规服务

**验收条件：**
- 每位员工卡片和详情页均有「邀请服务」CTA按钮
- 点击后引导至需求表单/报价流程
- 支持记录用户感兴趣的员工（埋点）

#### 2.2.4 设计语言

- **风格**：延续现有网站风格（毛玻璃效果、微渐变、圆角），但卡片信息层级更接近招聘平台
- **主题**：支持深色/浅色模式自动切换
- **动画**：卡片入场使用交错淡入效果（staggered fade-in），模拟简历列表浏览体验
- **薪资标签**：突出显示，吸引眼球，可用「省钱」风格徽章（如红色折扣标签样式）
- **中韩双语**：所有文本元素均配置中韩双语，默认显示韩文（韩企用户），可切换至中文

## 3. 验收标准

### 3.1 产品层验收（招聘市场场景）

#### P-AC-001：浏览团队列表（招聘集市浏览）

- **Given** 一位韩企合规负责人访问 /team 页面
- **When** 页面加载完成
- **Then** 页面展示6位数字员工的招聘风格信息卡片
- **And** 每位员工显示：中韩双语姓名、职位头衔、✓能力清单、经验数据、月薪及对标真人成本
- **And** 客户能快速浏览并定位到自己需要的合规方向

#### P-AC-002：中韩双语切换

- **Given** 用户在 /team 页面
- **When** 用户切换语言
- **Then** 所有文案（姓名、职位、能力清单、薪资描述）切换为对应语言
- **And** 切换过程无页面闪烁或内容丢失

#### P-AC-003：卡片交互与「面试冲动」

- **Given** 用户将鼠标悬停在某位员工卡片上
- **When** 悬停时间 ≥ 200ms
- **Then** 卡片微上浮（transform: translateY(-4px)），阴影加深
- **And** 薪资对比标签的视觉权重提升
- **And** CTA按钮颜色渐变变化

#### P-AC-004：员工详情查看（简历详情）

- **Given** 用户点击某位员工卡片
- **When** 触发点击事件
- **Then** 打开详情模态框（或跳转至详情页）
- **And** 展示完整能力清单、擅长场景示例、经验数据、薪资对比详情
- **And** 客户产生「请Ta为我服务」的冲动
- **And** ❌ 不展示神兽名称、灵魂来源、大师对标

#### P-AC-005：服务邀请引导

- **Given** 用户在员工详情页/卡片
- **When** 用户点击「邀请Ta为我服务」按钮
- **Then** 引导至需求表单/报价流程
- **And** 埋点记录用户感兴趣的员工

### 3.2 技术层验收

#### T-AC-001：性能
- 页面首屏加载时间 ≤ 1.5s（Lighthouse）
- 图片懒加载，员工头像使用 WebP 格式
- API 响应时间 ≤ 200ms（P95）

#### T-AC-002：可用性
- 支持 Chrome/Firefox/Safari/Edge 最新两个大版本
- 支持 iOS Safari / Android Chrome
- 离线状态显示友好提示

#### T-AC-003：可访问性
- 所有图片有 alt 文本
- 键盘导航支持（Tab 键遍历所有卡片）
- 对比度符合 WCAG 2.1 AA 标准

#### T-AC-004：安全
- API 接口无未授权访问
- 输入内容进行 XSS 过滤
- 埋点数据不包含 PII 信息

## 6. 合规审查

### 6.1 审查结论

**合规审查通过。签字：徐准，2026-05-05**

### 6.2 审查意见

| 审查维度 | 状态 | 说明 |
|---------|------|------|
| 数字员工身份标识 | ✅ 合规 | 全篇统一使用「数字员工」「AI数字员工」称谓，与真人明确区分 |
| 能力描述准确性 | ✅ 合规 | 能力清单限定在合规领域，经验数据标注清晰，无夸大 |
| 薪资对比合规性 | ⚠️ 关注 | 「对标真人成本」需注明数据来源和估算性质，避免误导用户以为AI具备真人执业资质 |
| 数据隐私安全 | ✅ 合规 | 明确API无未授权访问、XSS过滤、埋点不含PII |
| 用户引导合规 | ✅ 合规 | 服务邀请流程合理，后续需确保用户同意机制 |

### 6.3 合规建议

1. **薪资对比脚注**：在页面底部增加说明：「月薪对标数据基于2026年中国合规行业薪资调研，仅供参考。AI数字员工不替代持证律师/会计师的专业意见。」
2. **用户知情确认**：服务邀请流程中需加入数字员工服务条款的确认环节，确保用户知悉服务方为AI而非真人
3. **能力经验说明**：「经验：服务过XX+客户」应标注为基于训练数据的模拟经验，非真实服务经历

## 7. 法律审查

### 7.1 审查结论

**法律审查通过。签字：獬豸（法律合规官），2026-05-05**

### 7.2 审查意见

| 审查维度 | 状态 | 说明 |
|---------|------|------|
| 数字员工身份标识合法性 | ✅ 合规 | 全篇统一使用「数字员工」「AI数字员工」称谓，与真人服务明确区分，无诱导用户误认为真人律师的风险 |
| 能力描述法律风险 | ✅ 合规 | 能力清单限定在合规领域，无虚假宣传或超出服务能力的承诺 |
| 薪资对比法律风险 | ⚠️ 关注 | 「对标真人成本」标签需明确标注为估算数据，避免构成不正当竞争或价格误导。建议文案示例：「月薪 ¥999（行业参考：资深合规经理月薪约¥30,000+）」，增加「行业参考」前缀 |
| 「邀请服务」表述合规性 | ⚠️ 关注 | 按钮和流程中需增加明确声明：「本服务由AI数字员工提供，不构成正式法律意见」，避免用户误以为聘请了真人律师 |
| 用户协议/SLA覆盖 | ❌ 缺失 | PRD未包含用户协议、服务条款或SLA的设计。建议在开发阶段补充《数字员工服务条款》 |
| 经验数据法律效力 | ⚠️ 关注 | 「服务过30+韩企」等经验数据需明确为AI训练数据的模拟经验，非真实服务记录 |

### 7.3 法律建议

1. **服务邀请流程增加法律声明**：在服务邀请流程中加入固定提示，明确服务由AI提供，内容仅供参考，不构成正式法律意见。建议文案：「본 서비스는 AI 디지털 컴플라이언스 오피서가 제공하며, 공식 법률 자문을 대체하지 않습니다.（本服务由AI数字员工提供，不替代正式法律咨询。）」

2. **补充用户协议**：建议在Sprint 1或Sprint 2产出《数字员工服务条款》，包含但不限于：服务范围界定、免责条款、知识产权归属、数据使用说明、争议解决方式。

3. **薪资标签合规表述**：将「对标真人成本」改为更中性的表述，如「行业薪资参考」，并注明数据来源和估算性质。

4. **经验数据免责**：在详情页底部增加说明：「以上经验数据基于AI训练模型，为模拟经验展示，不构成真实服务承诺。」

5. **职级体系明确说明**：P10/P9/P8等职级仅代表AI数字员工的能力层级划分，建议在页面底部或帮助文档中说明此职级为内部能力分级，与真人律师的执业级别无关。

## 附录 A：设计参考与灵感

### 对标分析

| 产品 | 值得借鉴的点 | 我们如何超越 |
|------|------------|------------|
| 猎聘网候选人卡片 | 姓名+职位+技能标签+经验年限+薪资范围，信息密度高但清晰 | AI数字员工展示更简洁，且增加「薪资对比」独特卖点 |
| Deel /team | 展示全球专家团队，建立信任 | 我们的员工是AI原生，7x24在线，性价比突出 |
| 拉勾/Boss直聘职位卡片 | Headline亮眼，技能标签可点击筛选 | 数字员工的能力标签可作为筛选入口 |

### 设计原则

1. **招聘感优先**：用户第一眼应该觉得「这是一个人在招揽人才」，而不是「这是一个产品介绍页」
2. **性价比突出**：薪资对比标签是核心钩子，要让用户一眼看到「省了多少钱」
3. **专业感**：能力清单要具体、可验证（带年限/数量），避免空洞的形容词
4. **去神兽化**：所有山海经文化元素彻底删除，名称保留但只作为普通姓名展示
