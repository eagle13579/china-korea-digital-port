## 1. 产品概述

### 1.1 一句话定位
让韩企在线完成5~10道合规自检问答，自动生成中国市场合规健康度报告（PDF），触发销售跟进。

### 1.2 目标用户
| 用户类型 | 描述 |
|---------|------|
| 韩企CEO/代表理事 | 决策者，需要一个「快速了解我在中国市场合规状况」的工具 |
| 韩企法务/合规负责人 | 执行者，需要一份「可以拿给老板看」的正式合规评估报告 |
| 中国合作伙伴 | 中间人，需要一份「证明我们需要这个服务」的客观评估 |

### 1.3 痛点分析
- **用户不知道自己的合规缺口**：韩企对中国市场的合规要求知之甚少，但也不知道从何问起
- **销售入口太「重」**：现有咨询表单要求用户主动填写，心理门槛高
- **缺少「诱饵」**：一份免费的专业合规报告是比「免费咨询」更强的转化钩子
- **现有数据基础**：6位数字合规官（徐准、朴泰俊、丹书、李朴、金镇宇、崔敏智）覆盖全链路合规领域，完全有能力支撑自检工具

### 1.4 北极星关联
| 北极星维度 | 预期健康度提升 | 说明 |
|:----------|:--------------:|:-----|
| C商业化 | +10% | 自检工具是高转化率获客钩子 |
| B数字员工 | +5% | 自检结果关联到对应数字员工，直接展示价值 |

## 2. 功能规格

### 2.1 用户故事

#### US-001：完成合规自检问卷
**作为** 韩企CEO/合规负责人
**我想要** 在线完成一系列关于我企业中国业务的合规问题
**以便于** 快速评估我的合规风险等级

**验收条件：**
- 用户从头到尾完成5~10道选择题（单选/多选）
- 每道题有清晰的中韩双语问题描述
- 支持进度指示（第3/8题）
- 支持「上一步」回溯修改答案
- 无需登录即可开始（完成报告需留邮箱）
- 问卷在移动端和桌面端均适配良好

**题目设计（覆盖6位合规官领域）：**

| # | 领域 | 对应员工 | 问题示例 |
|:--|:-----|:---------|:---------|
| 1 | 市场准入 | 徐准 (P9) | 您的企业是否已完成中国市场的准入可行性评估？ |
| 2 | 投资架构 | 朴泰俊 (P10) | 您的企业是否已确立进入中国市场的投资架构（WFOE/JV/代表处）？ |
| 3 | 数据合规 | 丹书 (P8) | 您的企业是否涉及个人信息的跨境传输？ |
| 4 | 劳动用工 | 李朴 (P8) | 您的企业是否有中国境内的员工或外籍派遣员工？ |
| 5 | 法律合同 | 金镇宇 (P8) | 您的企业是否已审核与中国合作伙伴的商务合同？ |
| 6 | 公司注册 | 崔敏智 (P7) | 您的企业是否已完成中国子公司或分支机构的注册？ |
| 7 | 税务合规 | 朴泰俊 | 您的企业是否了解中国税收政策及跨境税务安排？ |
| 8 | 知识产权 | 金镇宇 | 您的企业在中国是否已完成商标/专利的知识产权布局？ |

#### US-002：获取合规健康度报告（PDF）
**作为** 韩企CEO
**我想要** 在完成自检后收到一份完整的合规健康度报告
**以便于** 了解我的合规短板和优先改进事项

**验收条件：**
- 问卷完成后，输入邮箱地址即可获取报告
- 报告通过邮件发送 + 页面直接下载
- 报告包含以下内容：
  - 总体合规评分（0-100分）及评级（优秀/良好/关注/危险）
  - 8个维度的雷达图评分
  - 每个维度的详细诊断结果
  - 针对低分维度的改善建议
  - 推荐对接的数字员工（如「您的数据合规方面需要关注，建议咨询丹书（数据合规官）」）
- 报告使用中韩双语（用户选择的语言）
- 报告头部包含企业名称、日期、报告编号
- 报告底部包含免责声明

#### US-003：报告触发的销售跟进
**作为** 销售团队（英招/鹿蜀）
**我想要** 在用户获取报告后自动创建一条线索
**以便于** 及时跟进高评分关注度的客户

**验收条件：**
- 用户提交邮箱获取报告时，自动创建一条线索到数据库中
- 线索包含：邮箱、企业名（如有）、合规评分、各维度评分、报告下载时间
- 新线索在admin后台显示（状态= 'new'，来源='合规自检'）
- 高评分关注度的客户（评分 ≤ 40分）标记为「高优先级」线索

### 2.3 API 设计

| 方法 | 路径 | 说明 | 优先级 |
|------|------|------|--------|
| GET | `/api/v1/compliance/questions` | 获取自检问卷题目 | P0 |
| POST | `/api/v1/compliance/submit` | 提交自检答案 | P0 |
| GET | `/api/v1/compliance/report/{token}` | 获取已生成的报告数据 | P0 |
| GET | `/api/v1/compliance/report/{token}/pdf` | 下载报告PDF | P0 |

#### 提交模型

```python
class ComplianceSubmission(BaseModel):
    answers: dict  # {question_id: answer_value}
    company_name: Optional[str] = None
    contact_name: Optional[str] = None
    email: str = Field(...)
    phone: Optional[str] = None
    language: str = "zh-CN"
```

#### 数据库新增表

```sql
CREATE TABLE IF NOT EXISTS compliance_checks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    token TEXT UNIQUE NOT NULL,
    answers TEXT NOT NULL,  -- JSON
    company_name TEXT,
    contact_name TEXT,
    email TEXT NOT NULL,
    phone TEXT,
    language TEXT DEFAULT 'zh-CN',
    score INTEGER,
    score_detail TEXT,  -- JSON
    report_generated INTEGER DEFAULT 0,
    report_downloaded INTEGER DEFAULT 0,
    status TEXT DEFAULT 'completed',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 评分算法

```
每个维度2分（共8个维度，满分16分，映射到100分制）：
- 每题选项分4级（0/0.5/1/2分）
- 总分 = (sum(各题得分) / 16) * 100
- 评级标准：
  - 优秀: ≥ 80分
  - 良好: 60-79分
  - 关注: 40-59分
  - 危险: < 40分
```

## 3. 验收标准

### 3.1 产品层验收

#### P-AC-001：完整自检流程
- **Given** 用户访问合规自检页面
- **When** 从第1题答题到第8题
- **Then** 每道题显示清晰，有进度指示
- **And** 可回退修改答案
- **And** 答题中途离开后重新进入可从断点继续（localStorage缓存）

#### P-AC-002：报告生成与下载
- **Given** 用户完成所有题目并提交邮箱
- **When** 点击「获取报告」
- **Then** 系统在 ≤ 5秒内生成报告并开始下载
- **And** 报告包含完整8维度评分、雷达图、改善建议
- **And** 报告为PDF格式，可直接打印

#### P-AC-003：销售线索自动创建
- **Given** 用户提交邮箱获取报告
- **When** 报告生成完成
- **Then** 系统在 admin 后台自动创建一条来源为「合规自检」的线索
- **And** 线索包含合规评分和维度详情

### 3.2 技术层验收

#### T-AC-001：性能
- 问卷页面首屏加载 ≤ 1.5s
- 报告生成 ≤ 5s（含PDF渲染）
- API响应时间 ≤ 300ms

#### T-AC-002：安全
- 报告使用随机token访问，不可遍历（UUID或nanoid）
- 邮箱格式校验
- 答案数据不包含PII（个人信息在answers外单独存储）
- PDF报告含防篡改水印

#### T-AC-003：可用性
- 支持Chrome/Firefox/Safari/Edge最新版
- 支持iOS Safari / Android Chrome
- 问卷离线状态下显示友好提示

## 4. 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| PDF生成在服务端消耗资源 | 中 | 中 | 先输出HTML报告（可直接打印PDF），服务端PDF生成用轻量库 |
| 8道题的评分算法可能不够全面 | 低 | 中 | 报告标注「初步评估」，建议专业深度评估请咨询数字员工 |
| 用户中途放弃率高 | 中 | 高 | localStorage缓存答案，支持断点续答；进度条减少焦虑 |
| 报告排名低分引起用户负面情绪 | 低 | 中 | 报告语气强调「改善建议」而非「问题」，标注AI生成仅供参考 |

## 6. 合规审查

### 6.1 审查结论
**合规审查通过。签字：徐准，2026-05-05**

### 6.2 审查意见
| 审查维度 | 状态 | 说明 |
|---------|------|------|
| 自检问题范围 | ✅ 合规 | 8道题覆盖核心合规领域，无不适当问题 |
| 报告评级表述 | ✅ 合规 | 评级标注「AI生成，仅供参考」，避免误导 |
| 邮箱收集合规 | ⚠️ 关注 | 需在报告页面增加「我已阅读并同意隐私政策」勾选框 |
| 数据跨境合规 | ✅ 合规 | 用户输入数据存储在中国境内服务器（阿里云） |
| 报告免责声明 | ✅ 合规 | 报告底部需包含明确的免责声明（已在模板中） |

### 6.3 合规建议
1. 报告免责声明建议写为：「本报告由AI数字员工基于您提供的信息自动生成，仅供一般性参考，不构成正式法律意见或合规建议。针对具体情况，建议咨询持证专业人士。」
2. 邮箱收集时增加隐私政策链接和同意勾选框
3. 报告保留期限建议在隐私政策中说明（建议6个月）

### 7.2 审查意见
| 审查维度 | 状态 | 说明 |
|---------|------|------|

### 7.3 法律建议
1. 在报告首页增加醒目的法律声明：「本报告由AI生成，不构成正式法律意见。对于重大合规决策，请咨询具备相应资质的专业人士。」
2. 在用户提交邮箱的界面增加「我已阅读并同意《隐私政策》」勾选框
3. 建议保留用户提交的数据和报告内容至少6个月，以应对潜在的争议

## 8. 不做清单

| 事项 | 原因 | 后续可能 |
|------|------|---------|
| ❌ 改为20+题深度问卷 | 用户完成率会大幅下降 | 可做深度版（v2） |
| ❌ 实时AI对话式自检 | 技术复杂度过高，MVP不需要 | v3可考虑集成大模型 |
| ❌ 上传文件分析（如上传公司章程） | 安全和合规风险高 | v2可考虑 |
| ❌ 生成的法律效力合规证书 | 不具备法律效力 | 不考虑 |
| ❌ 用户注册登录后再自检 | 增加摩擦，降低完成率 | 报告需留邮箱，已足够 |
| ❌ 保存用户历史报告 | 需要用户系统支撑 | PRD-008 (用户注册) 后考虑 |
| ❌ 行业对比基准（同行合规评分） | 数据量不足以支撑 | v3数据积累后考虑 |
| ❌ 报告在线分享功能 | 隐私风险，非核心需求 | v2如有需求可增加 |
| ❌ 自定义问卷（企业自己出题） | 完全偏离用途 | 不考虑 |
| ❌ 嵌入第三方CRM系统 | 超出MVP范围 | v3可考虑 |
