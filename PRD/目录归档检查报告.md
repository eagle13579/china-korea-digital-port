## 三、目录结构调整建议

### 建议1：创建 PRD/战略/产品架构文档.md

**原因**：SOP要求每个产品有 PRODUCT.md，包含合成来源、架构图、原子映射、北极星对齐。

**建议内容框架**：
```markdown
# 中韩出海数智港 · 产品架构文档
## 1. 合成来源（合规 + AI + 跨境）
## 2. 系统架构图（前端→FastAPI→SQLite→数字员工）
## 3. 北极星对齐（B数字员工 + C商业化）
## 4. 数字员工组织架构
## 5. 原子映射表（PRD/功能/数字员工）
```

### 建议2：补充对外版 README.md

**原因**：源码授权后交付给客户，需要产品介绍+部署教程+使用说明。

**建议内容框架**：
```markdown
# 中韩出海数智港 - China-Korea Digital Port
## 产品简介
## 快速开始（5分钟部署）
## License配置
## API文档
## 技术支持
```

### 建议3：统一合规数字员工文档位置

**当前问题**：`合规/数字员工/` 和 `PRD/合规/数字员工/` 内容重复。

**建议方案**：
- 保留 `PRD/合规/数字员工/` 作为正式位置
- 删除或软链 `合规/数字员工/`（避免歧义）

### 建议4：补充单元测试目录

**当前缺失**：无任何测试文件。

**建议方案**：
```
backend/
└── tests/
    ├── __init__.py
    ├── test_license.py     ← License校验测试
    ├── test_pricing.py     ← 定价API测试
    └── test_contact.py     ← 联系表单测试
```

### 建议5：部署前清理测试数据

**当前状态**：`backend/data/portal.db` 包含少量测试数据。

**建议**：发布前运行 `rm -f backend/data/portal.db`，让 init_db() 自动重建空的数据库。
