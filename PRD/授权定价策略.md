## 三、License授权机制

### 3.1 授权校验流程

```
客户购买 → 蠪侄(财务)确认收款
         → 孰湖(合同)签署授权协议
         → 英招(销售)获取客户信息
         → 调用 generate_license() 生成授权码
         → 通过邮件交付授权码
         → 客户配置到环境变量 CKDP_LICENSE 或 license.key 文件
         → 系统启动时 check_license() 校验
```

### 3.2 授权码结构

| 字段 | 说明 | 示例 |
|:-----|:------|:------|
| licensee | 被授权方名称 | 首尔贸易株式会社 |
| plan | 套餐类型 | annual |
| plan_name | 套餐名称 | 年订阅 |
| seats | 授权席位 | 5 |
| issued | 签发时间 | 2026-05-05T00:00:00+00:00 |
| expires | 过期时间 | 2027-05-05T00:00:00+00:00 |
| signature | 签名(防篡改) | a1b2c3d4e5f6g7h8 |

### 3.3 授权码生成

```python
from backend.license import generate_license

# 生成Business授权码（5席，1年有效）
key = generate_license(
    licensee="首尔贸易株式会社",
    plan="one-time",
    days=365,
    seats=5,
)
# 输出: Base64编码字符串
```

### 3.4 授权核验（启动时）

```python
from backend.license import check_license, format_license_status

status = check_license()
print(format_license_status(status))
if not status["valid"]:
    print("❌ 授权无效，请联系 admin@go-aiport.com")
    exit(1)
```

## 六、风险与应对

| 风险 | 应对措施 |
|:-----|:---------|
| License被破解 | 签名校验 + 定期轮换签名密钥 + 法律条款约束 |
| 定价过低影响品牌 | Enterprise档定位高端，避免低价低质印象 |
| 源码二次转售 | 协议禁止转售，OEM授权需白标 |
| 客户不续费 | 用年订阅锁客 + 持续功能迭代提升黏性 |
